# UX 改进设计方案（提案 · 待审）

> **目的**：让产品从"量化研究员仪表盘"转向"交易员的盘中 / 收盘决策助手"——更直接、更人性化。
> **铁律边界**：引擎冻结、`LLM只润色不判断`、`strategy_hash不可覆盖`。
> - 第 1 层（纯展示 / 模板）：**不触碰引擎**，符合铁律。
> - 第 2 层（量价进信号）：**触及冻结边界**，需你明确放宽后才做。
> **诊断依据**：基于实际代码（非印象），见 §1。现状读取自 `MarketOverview.vue` / `EtfDetail.vue` / `opinion_engine/templates.py` / `strategy_engine/engine.py`。

---

## 1. 现状诊断（基于代码）

### 1.1 用户现在实际看到什么
- **主看板** `frontend/src/views/MarketOverview.vue`：指数涨跌条 + 涨跌家数 + 风险汇总 + 一张 19 行信号表；档位（"允许小仓位试错"）只是表格里一个**小徽章**。
- **详情页** `frontend/src/views/EtfDetail.vue`：那句人话（"512760｜允许小仓位试错（综合72分/置信70%）…"）被放在页面**底部**的"盘中 / 复盘意见"卡片，经 `OpinionList` 渲染。
- **后端有 `phase` 字段**（`pre_market`/`midday`/`pre_close`/`post_close`），但前端**完全没用它做模式切换**（grep 确认无 vue/ts 按 phase 改变体验）。
- **引擎未把"量价"做成信号**：`engine.py:61` 仅把 `volume` 作为原始字段存储，信号完全来自价格类指标（MA20 突破 / 斜率 / RSI / 相对强弱 / 板块趋势）。你原始需求第一句"盘中根据**量价关系**"目前**没进信号**。

### 1.2 与原始需求的结构性偏差
原始需求："盘中根据**量价关系** + **涨跌幅度**给交易建议；收盘后复盘。"
- **量价**：引擎未实现（见 1.1）。→ 原始核心需求未落地。
- **涨跌幅度**：快照有 `涨跌幅`，部分规则可能用到，但非"量价配合"语义。
- **双模式**：后端分了 phase，前端未分化体验。

### 1.3 为什么"不直白 / 不人性化"（归因）
| 维度 | 现状 | 用户预期 |
|---|---|---|
| 结论位置 | 仪表盘，行动藏表格徽章 + 详情页底部 | 一眼看到"今天该买 / 卖什么" |
| 模式 | 单一，无视盘中 / 收盘 | 盘中给动作、收盘给复盘 |
| 量价 | 缺（仅存原始字段） | 核心输入 |
| 信任 | 全"置信 70%"显眼 | 行动建议压过置信 |

## 2. 设计原则
1. **引擎冻结不可破**（第 1 层前提）：只动展示层 + 意见模板文案，不动评分数学。
2. **LLM只润色不判断**：人话文案由确定性模板生成（`opinion_engine/templates.py`），不引入外部判断。
3. **结论前置**：把"行动"放到第一屏，指标作为"依据"放在下方。
4. **双模式分流**：盘中 = 此刻动作（买 / 卖 / 等）；收盘 = 复盘（触发 / 失效 / 观察）。
5. **渐进披露**：概览给结论，详情给依据；不一次砸给 19 张图。

## 3. 第 1 层：纯前端快速改进（不碰引擎）

### 3.1 信息架构（新）
```
市场总览
├─ [今日关注榜]            ← 新增，置顶（结论前置）
├─ 盘中 / 收盘复盘  [Tab]   ← 新增（双模式分流）
├─ 主要指数 / 市场宽度 / 信号风险汇总   （保留）
└─ 最新信号表              （保留，作为完整清单）
```

### 3.2 今日关注榜（核心新增）
- **取数**：`/api/signals/latest` 全量，前端按 `TIER_ORDER`（积极度）降序。
- **筛选**：仅展示可操作档位 `OPPORTUNITY_ENHANCE` + `SMALL_POSITION`；其余折叠为一行"观察 N 只 / 规避 N 只"。
- **每条卡片**：ETF 名 + 档位徽章（大字）+ 一句人话（复用 `opinion.content` 或 `key_metrics_text`）+ 建议仓位区间。
- **文案示例**：`芯片ETF ｜ 可小仓试错 ｜ 半导体板块突破20日线、相对大盘走强 ｜ 10–25%`
- **排序**：同档位内按 `score` 降序；总量控制在 ≤5 条，避免信息过载。

### 3.3 盘中 / 收盘复盘 双 Tab
- 用 `signal.phase` / `opinion.phase`（后端已有）区分；前端加 `mode` 状态，默认按当前北京时间推断（盘中 vs 收盘后）。
- **盘中 Tab**：强调"此刻动作"——关注榜 + 量价 / 涨跌快照 + 风险等级；文案偏"现在"。
- **收盘复盘 Tab**：强调"今日复盘"——哪些信号触发、哪些失效（`invalidation_conditions`）、明日观察清单；可复用 `/api/signals/history` 做日对比。

### 3.4 详情页人话置顶
- `EtfDetail.vue`：把 `latest_signal` 的 `signal_type_text` + `position_text` + 一句 `opinion.content` 提到最上方 **Hero 区（大字）**，图表与历史下放。
- `OpinionList` 每条：标题（`opinion.title`）+ 内容（人话）+ **触发规则 / 失效条件作为"依据"折叠区**（渐进披露）。

### 3.5 置信度措辞弱化
- `fmtConfidence` 仍计算，但 UI 上降为次级信息（小字 / 灰色），或映射为三档"高 / 中 / 低"（70% 统一显示"中"），避免压过行动建议。
- 说明：70% 上限因板块资金流缺（D4）所致，属数据限制；可在关注榜底部加一行："板块资金流暂缺，置信度保守"。

### 3.6 涉及文件与改动量
| 文件 | 改动 |
|---|---|
| `MarketOverview.vue` | 加关注榜 + 双 Tab 壳 |
| `components/sections/WatchBoard.vue`（**新**） | 今日关注榜 |
| `components/sections/OpinionList.vue` | 人话置顶、依据折叠 |
| `EtfDetail.vue` | Hero 区人话置顶 |
| `lib/format.ts` | 置信度三档映射（可选） |
| `opinion_engine/templates.py` | 可选：强化 `key_metrics_text` 中文（"放量"等需第 2 层量价数据） |
| **预计** | **1–2 天，纯前端 / 模板，不碰引擎** |

## 4. 第 2 层：量价进信号（需拍板，触冻结边界）

### 4.1 缺口
用户原始需求"量价关系"未实现。当前 `volume` 仅存原始字段，无"放量 / 缩量 / 量价背离"判定。

### 4.2 方案 A：仅展示（不进评分，最保守）
- 前端用已存的 `volume` / `amount` 画"量价配合"图（成交量柱 + 价格叠加，标注放量日）。
- **不进 `strategy_hash`，零风险**。但信号本身不含量价，仍是"价格建议 + 量价看图"。

### 4.3 方案 B：量价进信号（需放宽冻结）
- **新增确定性量价指标（additive，不调权重）**：
  - `vol_ratio` = 今日量 / N 日均量（放量倍数）
  - `vol_price_diverge` = 价格新高但量未新高（量价背离）
  - `vol_breakout` = 突破日放量（量价齐升确认）
- **新增 `triggered_rules`**（如 `VOL_BREAKOUT` / `VOL_PRICE_DIVERGE`），仅作为 `supporting_metrics` + 触发规则补充，**不修改现有权重 / score 公式**。
- **`strategy_hash` 影响**：新增指标会改变 hash 语义 → 必须明确"扩展而非覆盖"，并升版 `template_version` / `metric_definition_version`。
- **边界建议**：若坚持"strategy_hash不可覆盖"绝对化 → 只做方案 A；若要真实现量价建议 → 必须接受"扩展"并文档化。

### 4.4 建议
先第 1 层（立刻见效，零风险）。第 2 层作为独立任务，默认**方案 A（展示）**；方案 B 需你明确放宽冻结边界后再做，且只"新增指标不调权重"。

---

## 5. 落地里程碑
1. **第 1 层（纯前端）** → 1–2 天 → 直白度显著提升，置信 / 双模式问题解决。
2. **第 2 层方案 A（量价展示）** → +0.5 天 → 用户能看图自行判断量价。
3. **第 2 层方案 B（量价进信号）** → 待拍板 → 信号真正含量价。

- [ ] 有"盘中 / 收盘"切换且内容明显不同。
- [ ] 详情页第一屏是人话结论，不是图表。
- [ ] 置信度不压过行动建议。
- [ ] （第 2 层后）量价关系在建议中有体现。
